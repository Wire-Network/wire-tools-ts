# wire-test-cluster: Local Setup (Non-Docker)

End-to-end instructions for building Wire's `wire-test-cluster` CLI from source on the host (Ubuntu 24.04 / WSL2), **without** the Docker image. Based on the five repos' own README / CLAUDE.md files; the reference Dockerfile is out of date in a few places (see "Differences from the Dockerfile" at the bottom).

> **Status:** built and smoke-tested on Ubuntu 24.04 WSL2, 2026-04-24.
> End-to-end `create` + `run` both succeed â€” cluster spins up 1 bios + 5
> producer + 3 batchop + 1 uwrit `nodeop` processes, `anvil` with OPP
> contracts, and `solana-test-validator` with the `opp_outpost` BPF
> program preloaded.

## 2. Environment

```bash
export WIRE_ROOT="$HOME/ext"        # parent directory of all repos â€” adjust if different
export CC=/usr/bin/clang-18
export CXX=/usr/bin/clang++-18
# Strip any linuxbrew paths to prevent library conflicts
export PATH=$(echo "$PATH" | tr ':' '\n' | grep -v linuxbrew | tr '\n' ':' | sed 's/:$//')
# Keep a few cores free so 4GB-per-file compilation units don't OOM
export NUM_JOBS=$(( $(nproc) - 2 ))
```

## 3. Build steps

> Each step assumes `cd "$WIRE_ROOT/<repo>"` unless stated otherwise.

### 3.1 `wire-libraries-ts` â€” shared JS/TS packages + protoc plugins

This produces the `@wireio/*` packages and the three protoc CLI binaries
(`wire-protobuf-bundler`, `protoc-gen-solidity`, `protoc-gen-solana`) that
`generate-opp-bundles.fish` needs on PATH.

```bash
cd "$WIRE_ROOT/wire-libraries-ts"
pnpm install                      # postinstall runs pnpm dist for protoc-gen-* (slow first time)
pnpm build

# Put the three CLI binaries on PATH for the OPP generation step:
for p in protoc-gen-solidity protoc-gen-solana protobuf-bundler; do
  (cd "packages/$p" && pnpm run dist && pnpm link --global)
done

# Verify:
command -v wire-protobuf-bundler protoc-gen-solidity protoc-gen-solana
```

**Note on `pnpm link --global`:** Only the three CLI packages need this, and
only so the `.fish` script finds the binaries on PATH. `@wireio/shared`,
`@wireio/shared-node`, `@wireio/sdk-core`, etc. are resolved automatically by
`wire-e2e-tests/.pnpmfile.cjs` â€” **do not** manually link them.

### 3.2 `wire-cdt` â€” Contract Development Toolkit (WASM/ABI compilers)

Required by `wire-sysio` so `BUILD_SYSTEM_CONTRACTS=ON` can build the `.wasm`
contracts. Installs into a local prefix we point `wire-sysio` at.

```bash
cd "$WIRE_ROOT/wire-cdt"
./vcpkg/bootstrap-vcpkg.sh       # one-time

cmake -S . -B build/debug -G Ninja \
    -DCMAKE_BUILD_TYPE=Debug \
    -DENABLE_CCACHE=ON \
    -DENABLE_TESTS=ON \
    -DCMAKE_C_COMPILER="$CC" \
    -DCMAKE_CXX_COMPILER="$CXX" \
    -DCMAKE_TOOLCHAIN_FILE="$PWD/vcpkg/scripts/buildsystems/vcpkg.cmake" \
    -DCMAKE_INSTALL_PREFIX="$WIRE_ROOT/wire-install" \
    -DCMAKE_PREFIX_PATH="$WIRE_ROOT/wire-install"

cmake --build build/debug -j"$NUM_JOBS"
cmake --install build/debug
```

### 3.3 `wire-sysio` â€” node software (`nodeop`, `kiod`, `clio`, `sys-util`)

> **Constraint from `CLAUDE.md`:** the build directory MUST be under
> `wire-sysio/build/<name>` (e.g. `build/claude`, `build/debug-clang-clion`).
> Other locations break `configure_file` for test reference data.

```bash
cd "$WIRE_ROOT/wire-sysio"
./vcpkg/bootstrap-vcpkg.sh       # one-time

export BUILD_DIR="$PWD/build/debug"
cmake -S . -B "$BUILD_DIR" -G Ninja \
    -DCMAKE_BUILD_TYPE=Debug \
    -DBUILD_SYSTEM_CONTRACTS=ON \
    -DBUILD_TEST_CONTRACTS=ON \
    -DENABLE_CCACHE=ON \
    -DENABLE_TESTS=ON \
    -DCMAKE_C_COMPILER="$CC" \
    -DCMAKE_CXX_COMPILER="$CXX" \
    -DCMAKE_TOOLCHAIN_FILE="$PWD/vcpkg/scripts/buildsystems/vcpkg.cmake" \
    -DCMAKE_INSTALL_PREFIX="$WIRE_ROOT/wire-install" \
    -DCMAKE_PREFIX_PATH="$WIRE_ROOT/wire-install/cdt"

ninja -C "$BUILD_DIR" opp_cdt_models.protos   # see GOTCHA #1 below
# Make sure the protoc CLIs are on PATH â€” there's a POST_BUILD step on
# libopp.a that invokes generate-opp-bundles.fish (GOTCHA #2 below).
export PATH="$HOME/.local/share/pnpm:$PATH"
cmake --build "$BUILD_DIR" -j"$NUM_JOBS"
# Some compilation units need ~4GB RAM; reduce -j if you see OOMs.
```

> **GOTCHA â€” first-build race on the zpp-generated contract headers.** The
> `contracts_project` external target depends on `opp_cdt_models.protos`
> (which runs `cdt::protoc-gen-zpp` to generate `sysio/opp/**/*.pb.hpp` in
> `build/debug/libraries/opp/generated-cdt/`). With Ninja, that dependency
> doesn't reliably fire on a cold build â€” the contracts build runs before
> the `.pb.hpp` files exist and blows up with
> `fatal error: 'sysio/opp/types/types.pb.hpp' file not found`. Running
> `ninja opp_cdt_models.protos` explicitly before the full build generates
> the headers, after which the main `cmake --build` proceeds cleanly.

### 3.4 Generate the `wire-opp/` sibling directory

`wire-opp` is **not** a repo â€” it's generated from the proto definitions in
`wire-sysio/libraries/opp/proto/` via a fish script, and its output is
consumed by `wire-ethereum` and `wire-e2e-tests` through their
`.pnpmfile.cjs` hooks.

```bash
cd "$WIRE_ROOT/wire-sysio/libraries/opp/tools"
./generate-opp-bundles.fish
# Creates $WIRE_ROOT/wire-opp/{typescript,solidity,solana}/
```

Prerequisites: the three CLIs from step 3.1 must be on PATH. The script checks and aborts with a clear error otherwise.

### 3.5 `wire-ethereum` â€” Solidity contracts (Hardhat)

wire-ethereum uses **npm**, not pnpm â€” the `.pnpmfile.cjs` auto-linking does
not apply here. Link the generated `@wireio/opp-*` packages globally into
npm's global bin first, then link them into the project:

```bash
# One-time: register the wire-opp/* packages as globally-linkable via npm.
(cd "$WIRE_ROOT/wire-opp/solidity"   && npm link)
(cd "$WIRE_ROOT/wire-opp/typescript" && npm link)

cd "$WIRE_ROOT/wire-ethereum"
git checkout feature/protobufs-for-opp      # see Â§1.4
npm install
npm link @wireio/opp-solidity-models @wireio/opp-typescript-models
npx hardhat compile
```

### 3.5b Anchor â€” build `opp-outpost` in `wire-solana`

The harness loads this Solana program onto `solana-test-validator` via
`--bpf-program`, and reads its IDL for CPI. Requires `anchor-cli 0.31.0`.

```bash

# (or via avm: cargo install --git https://github.com/coral-xyz/anchor avm --locked && avm install 0.31.0 && avm use 0.31.0)

# Produces target/deploy/opp_outpost.so and target/idl/opp_outpost.json
```

### 3.6 `wire-e2e-tests` â€” harness monorepo + `wire-test-cluster` CLI

```bash
cd "$WIRE_ROOT/wire-e2e-tests"
pnpm install            # .pnpmfile.cjs auto-links @wireio/* from ../wire-libraries-ts and ../wire-opp
pnpm build              # tsc -b composite build across flow-aâ€¦flow-d + harness

# Expose wire-test-cluster on PATH â€” the harness package.json declares
# "bin": { "wire-test-cluster": "lib/cjs/cli.js" }, so a plain link

```

## 4. Running the cluster

```bash
# Kill any leftover chain processes from a previous run.
# Use exact names â€” an argv regex like "nodeop|anvil" can match the calling
# shell itself and SIGTERM your own session (exit 144).
killall nodeop kiod anvil solana-test-validator 2>/dev/null || true

export CHAIN_DIR="$HOME/ext/wire-chains/dev-001"

wire-test-cluster \
    --cluster-path="$CHAIN_DIR" \
    --force \
    create \
        --build-path="$WIRE_ROOT/wire-sysio/build/debug" \
        --ethereum-path="$WIRE_ROOT/wire-ethereum" \
        --solana-path="$WIRE_ROOT/wire-solana" \
        --prod-count=5 \
        --pnodes=1 \
        --batch-operators=3 \
        --underwriters=1 \
        --epoch-duration=60

wire-test-cluster --cluster-path="$CHAIN_DIR" run
# Ctrl+C to stop. To wipe: wire-test-cluster --cluster-path="$CHAIN_DIR" destroy
```

> Note: the harness CLI looks for `nodeop`/`kiod`/`clio`/`sys-util` at
> `<build-path>/bin/<name>`. A full successful `cmake --build` populates
> that directory automatically via the `copy_bin` macro. If `build/bin/`
> is missing the binaries, the build didn't actually finish; check the
> ninja log for earlier failures (e.g. the OPP POST_BUILD fish gotcha in
> Â§6.5).

Flag reference (from `wire-e2e-tests/packages/harness/src/cli.ts`):

| Flag | Scope | Notes |
|---|---|---|
| `--cluster-path` | global, required | Directory for chain data, config, logs |
| `--force` | global | Remove existing cluster-path before `create` |
| `--build-path` | create, required | wire-sysio build directory |
| `--ethereum-path` | create | wire-ethereum root; bootstraps anvil + OPP contracts |
| `--solana-path` | create | wire-solana root; bootstraps solana-test-validator |
| `-p/--pnodes` | create | Producer node count |
| `-n/--nodes` | create | Non-producer node count |
| `--prod-count` | create | Producers to register (default 21) |
| `--batch-operators` | create | 3â€“21 |
| `--underwriters` | create | 1â€“100 |
| `--epoch-duration` | create | Seconds |

## 5. Differences from the reference Dockerfile

Noted because the Dockerfile in circulation is partly stale:

- **`wire-opp` layout.** Dockerfile uses `${WIRE_ROOT}/wire-opp` and assumes
  it appears as a side-effect of the `wire-sysio` build. It does not â€” you
  must run `generate-opp-bundles.fish` explicitly.
- **`pnpm link --global` for shared/sdk-core.** Dockerfile globally-links
  `shared`, `shared-node`, `shared-web`, `sdk-core`. Not needed locally â€”
  `wire-e2e-tests/.pnpmfile.cjs` hooks resolve them from
  `../wire-libraries-ts/packages/` automatically. The only packages that
  still need a global link are the three CLI binaries (so `.fish` finds them).
- **Node 24 vs Node 22.** Dockerfile pins 24.14.1; the monorepos' `engines`
  only require `>=22`, and v22 works fine.
- **wire-ethereum `npm i` of pinned opp-models.** Dockerfile does
  `npm i && npm link @wireio/opp-solidity-models @wireio/opp-typescript-models`
  to work around a `package-lock.json` that pins 1.0.9 (missing types). Still
  applies locally.
- **A line in the Dockerfile is a bug:**
  `RUN test -d ${WIRE_OPP_ROOT}/typescript && ${WIRE_OPP_ROOT}/solidity && â€¦`
  executes `${WIRE_OPP_ROOT}/solidity` as a command instead of checking the
  directory. It would silently fail; we don't replicate this.

## 6. Troubleshooting / gotchas hit during setup

### 6.1 `@yao-pkg/pkg-fetch` race during `pnpm install` in `wire-libraries-ts`

Two `protoc-gen-*` postinstall scripts run `pnpm dist` in parallel, both
calling `pkg-fetch` which writes to the same `~/.pkg-cache/v3.5/` â€” one wins,
the other fails with `ENOENT: ... fetched-v22.22.0-linux-x64.downloading`.
**Fix:** pre-populate the cache once before `pnpm install`:

```bash
npx --yes @yao-pkg/pkg-fetch node22 linux x64     # fills ~/.pkg-cache/v3.5/
```

### 6.2 `pnpm link --global` errors with `ERR_PNPM_NO_GLOBAL_BIN_DIR`

Fresh pnpm installs don't pick up a global bin dir. Set it once:

```bash
pnpm config set global-bin-dir /home/<you>/.local/share/pnpm
pnpm config set global-dir     /home/<you>/.local/share/pnpm/global
# and add PNPM_HOME to your shell rc:
export PNPM_HOME="$HOME/.local/share/pnpm"
export PATH="$PNPM_HOME:$PATH"
```

### 6.3 `protoc-gen-ts not found` during `generate-opp-bundles.fish`

`wire-protobuf-bundler` calls `protoc-gen-ts` from `@protobuf-ts/plugin`
(not the 3 CLIs we built from `wire-libraries-ts`). Install globally:

```bash
pnpm add -g @protobuf-ts/plugin     # provides protoc-gen-ts on PATH
```

### 6.4 GOTCHA #1 â€” `opp_cdt_models.protos` not generated before `contracts_project` build

`cmake --build build/debug` on a fresh build fails:

```
fatal error: 'sysio/opp/types/types.pb.hpp' file not found
```

The `contracts_project` ExternalProject kicks off before Ninja generates the
zpp `.pb.hpp` headers, despite `add_dependencies(contracts_project
opp_cdt_models.protos)`. Work around by explicitly generating first:

```bash
ninja -C build/debug opp_cdt_models.protos
cmake --build build/debug -j${NUM_JOBS}
```

### 6.5 GOTCHA #2 â€” POST_BUILD step on libopp.a always fires, even when CLIs aren't on PATH

`libraries/opp/CMakeLists.txt` checks for the required CLIs via a
`find_program` loop, but reuses the same cache variable (`_found_prog`) for
every iteration â€” after the first `find_program` populates it, the
subsequent calls are no-ops. The "tools missing" warning branch therefore
never fires and the `fish generate-opp-bundles.fish` POST_BUILD step is
attached to `libopp.a` unconditionally. If `wire-protobuf-bundler` /
`protoc-gen-solidity` / `protoc-gen-solana` aren't on `PATH` during the
build, the link step fails with:

```
Error: The following required tools are not found on PATH:
  - wire-protobuf-bundler
  - protoc-gen-solidity
  - protoc-gen-solana
```

Fix: ensure `PNPM_HOME/bin` is on `PATH` during `cmake --build`. Long-term,
the CMakeLists.txt should pass a fresh output variable per iteration (or
use `find_program(... REQUIRED)`).

### 6.5 `wire-test-cluster create` crashes with `paths[0] must be string` when `--solana-path` is omitted

Harness `cli.ts:233` calls `Path.resolve(argv.solanaPath)` unconditionally.
Until fixed in harness, always pass `--solana-path=<wire-solana-root>` even
if the Solana program isn't deployed â€” `SOLBootstrap` will warn and skip
program init when the expected `<wire-solana>/wallets/opp-outpost-keypair.json`
is absent.

### 6.6 `wire-solana` master is the liqSol repo â€” use the OPP branch

`wire-solana` on `master` is the Capital Staking / liqSol program and does
not contain `opp-outpost`. Checkout
`feature/opp-solana-outpost-integration`, which adds `programs/opp-outpost/`,
`wallets/opp-outpost-keypair.json`, and the Anchor.toml entry. Then
`anchor build -p opp-outpost` produces the `.so` + IDL the harness expects.

### 6.7 `pkill -f "<regex>"` self-terminates your session

The Dockerfile's run instructions suggest
`pkill -f "solana-test-validator|anvil|kiod|nodeop"`. Because that regex is
matched against the full argv (including the pattern string itself), `pkill`
kills its parent shell â€” the Bash/zsh invocation exits 144 and `wire-test-cluster`
never runs. Use `killall <exact name>` instead.
